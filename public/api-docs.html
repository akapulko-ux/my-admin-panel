Аутентификация

Для доступа к API необходимо использовать API ключ. Ключ должен передаваться в заголовке запроса.

Заголовки аутентификации

X-API-Key: sk_live_your_api_key_here

Пример запроса

curl -H "X-API-Key: sk_live_your_api_key_here" \
  https://us-central1-bali-estate-1130f.cloudfunctions.net/api/v1/fixations

Лимиты запросов

API имеет ограничения на количество запросов в зависимости от роли пользователя.

Роль Лимит запросов
Админ 10,000
Премиум агент / Премиум застройщик 5,000
Агент / Застройщик 1,000

Фиксации клиентов

GET /fixations

Получение списка фиксаций клиентов с поддержкой фильтрации и пагинации.

Параметры запроса

  • status - Фильтр по статусу (pending, approved, rejected, expired)
  • startDate - Дата начала (YYYY-MM-DD)
  • endDate - Дата окончания (YYYY-MM-DD)
  • limit - Количество записей на странице (по умолчанию: 20)
  • page - Номер страницы (по умолчанию: 1)

Пример ответа

{
  "data": [
    {
      "id": "fixation_id",
      "clientName": "Иван Иванов",
      "clientPhone": "+79001234567",
      "agentName": "Агент Петров",
      "complexName": "ЖК Солнечный",
      "developerName": "ООО Застройщик",
      "propertyType": "Квартира",
      "status": "pending",
      "dateTime": "2024-01-15T10:30:00Z",
      "validUntil": "2024-01-22T10:30:00Z"
    }
  ],
  "pagination": {
    "page": 1,
    "limit": 20,
    "total": 150,
    "totalPages": 8
  }
}

Webhook

Как работают Webhook

Webhook позволяют получать уведомления в реальном времени о событиях в системе. При наступлении события система отправляет HTTP POST запрос на указанный URL.

Поддерживаемые события

  • fixation.created - Создана новая фиксация
  • fixation.updated - Обновлена фиксация
  • fixation.status_changed - Изменен статус фиксации
  • fixation.expired - Истек срок фиксации
  • fixation.rejected - Фиксация отклонена

Webhook эндпоинты

GET /webhooks

Получение списка webhook подписок

POST /webhooks

Создание новой webhook подписки

POST /webhooks/{id}/test

Тестирование webhook подписки

Пример webhook payload

{
  "event": "fixation.created",
  "timestamp": "2024-01-15T10:30:00Z",
  "data": {
    "fixationId": "fixation_id",
    "clientName": "Иван Иванов",
    "clientPhone": "+79001234567",
    "agentName": "Агент Петров",
    "complexName": "ЖК Солнечный",
    "developerName": "ООО Застройщик",
    "propertyType": "Квартира",
    "status": "pending",
    "dateTime": "2024-01-15T10:30:00Z"
  }
}

Заголовки webhook

  • Content-Type: application/json
  • X-Webhook-Signature: hmac_sha256_signature
  • User-Agent: IT-Agent-API/1.0

Обработка ошибок

API возвращает стандартные HTTP коды состояния и JSON объекты с описанием ошибок.

400 Bad Request

Некорректный запрос или неверные параметры

401 Unauthorized

Отсутствует или неверный API ключ

403 Forbidden

Недостаточно прав для доступа к ресурсу

404 Not Found

Запрашиваемый ресурс не найден

429 Too Many Requests

Превышен лимит запросов

500 Internal Server Error

Внутренняя ошибка сервера

Пример ответа с ошибкой

{
  "error": "Invalid API key",
  "message": "The provided API key is not valid or has expired"
}
